Implementation: Define a Performance Metric¶

It is difficult to measure the quality of a given model without quantifying its performance over training and testing. This is typically done using some type of performance metric, whether it is through calculating some type of error, the goodness of fit, or some other useful measurement. For this project, you will be calculating the coefficient of determination, R2, to quantify your model's performance. The coefficient of determination for a model is a useful statistic in regression analysis, as it often describes how "good" that model is at making predictions.

The values for R2 range from 0 to 1, which captures the percentage of squared correlation between the predicted and actual values of the target variable. A model with an R2 of 0 is no better than a model that always predicts the mean of the target variable, whereas a model with an R2 of 1 perfectly predicts the target variable. Any value between 0 and 1 indicates what percentage of the target variable, using this model, can be explained by the features. A model can be given a negative R2 as well, which indicates that the model is arbitrarily worse than one that always predicts the mean of the target variable.

# Import 'r2_score'
from sklearn.metrics import r2_score

def performance_metric(y_true, y_predict):
    """ Calculates and returns the performance score between 
        true and predicted values based on the metric chosen. """
    
    # Calculate the performance score between 'y_true' and 'y_predict'
    score = r2_score(y_true, y_predict)
    
    # Return the score
    return score

Question 7 - Grid Search¶

  • What is the grid search technique?
  • How it can be applied to optimize a learning algorithm?

Hint: When explaining the Grid Search technique, be sure to touch upon why it is used, what the 'grid' entails and what the end goal of this method is. To solidify your answer, you can also give an example of a parameter in a model that can be optimized using this approach.

Answer:

The algorithm of a learning model chooses the weights that minimizes training error. But this process takes as given the structure of the model. For polynomial regression, the structure of the model could be determined by the polynomial degree . For decision trees, structure is determined by the depth of the tree. Grid search adds another layer to the learning process. For each preset values of the parameters that determines the structure of the model (say, max depth 1, 2, 3, 4,...,10 for the decision tree), the model is trained and the cross validation error is calculated. The grid search technique then chooses the model structure that yields the lowest validation score.

Question 8 - Cross-Validation¶

  • What is the k-fold cross-validation training technique?

  • What benefit does this technique provide for grid search when optimizing a model?

Hint: When explaining the k-fold cross validation technique, be sure to touch upon what 'k' is, how the dataset is split into different parts for training and testing and the number of times it is run based on the 'k' value.

When thinking about how k-fold cross validation helps grid search, think about the main drawbacks of grid search which are hinged upon using a particular subset of data for training or testing and how k-fold cv could help alleviate that. You can refer to the docs for your answer.

Answer:

Evaluating the model based only on training error may result in overfitting. Training error will typically be always lower than the error evaluated using any other data. This is because the learning algorithm's objective is to find weights that minimize training error. To avoid this, the model must be evaluated using data that it has not learned. This data--called the test set--is typically a subset of the original data set aside exactly for this purpose.

However, the learning algorithm only seeks to find the optimal weights for a given structure of the model. This structure (e.g. degree of the polynomial, maximum depth) must be chosen by the engineer (or using grid search) based on some metric calculated using data excluded from training. One could use the test data for this, but finding the optimal structure based on performance in the test data again introduces overfitting. You can no longer evaluate the final model using the testing data because the model has now "learned" it when you chose the model's structure based on a metric evaluated using the test set.

A third hold out sample must be used to find the optimal model structure. This is the validation set. But now our original data must be split into three subsets: a training set, a validation set, and a testing set. This may not be a problem if our original data is much larger than what is required for the level of complexity required by our problem (a 10th degree polynomial model requires significantly more data than a simple linear model to avoid overfitting). For smaller datasets, a different technique must be used. This technique is called k-fold cross-validation.

k-fold cross-validation works as follows:

1) Shuffle the entire dataset and set aside a test set for final testing

2) Split the remaining data into $k$ equal-sized folds.

3) Train the model on the remaining $N - n_k$ data and calculate its validation score on the remaining $n_k$ sample

4) Repeat step 3) for each $k$.

5) Take the average of each of the $k$ validation scores. This average will be the validation score for the given structure of the model.

The advantage of this approach over setting aside just one validation set is that all of the data (minus the test set) can be used for training. By having more data for training, the likelihood of overfitting has been reduced. The grid search technique can now use the validation error to determine the optimal structure of the model.

# Import 'make_scorer', 'DecisionTreeRegressor', and 'GridSearchCV'
from sklearn.tree import DecisionTreeRegressor
from sklearn.metrics import make_scorer
from sklearn.grid_search import GridSearchCV

def fit_model(X, y):
    """ Performs grid search over the 'max_depth' parameter for a 
        decision tree regressor trained on the input data [X, y]. """
    
    # Create cross-validation sets from the training data
    # sklearn version 0.18: ShuffleSplit(n_splits=10, test_size=0.1, train_size=None, random_state=None)
    # sklearn versiin 0.17: ShuffleSplit(n, n_iter=10, test_size=0.1, train_size=None, random_state=None)
    cv_sets = ShuffleSplit(X.shape[0], n_iter = 10, test_size = 0.20, random_state = 0)

    # Create a decision tree regressor object
    regressor = DecisionTreeRegressor()

    # Create a dictionary for the parameter 'max_depth' with a range from 1 to 10
    params = {'max_depth': [n+1 for n in range(10)]}

    # Transform 'performance_metric' into a scoring function using 'make_scorer' 
    # make_scorer is just a wrapper that decorates your performance metric and turns it into
    # a scoring function (if your performance metric is an error, it flips the sign to turn
    # it into a score)
    scoring_fnc = make_scorer(performance_metric)

    # Create the grid search cv object --> GridSearchCV()
    # Make sure to include the right parameters in the object:
    # (estimator, param_grid, scoring, cv) which have values 'regressor', 'params', 'scoring_fnc', and 'cv_sets' respectively.
    grid = GridSearchCV(regressor, params, scoring=scoring_fnc, cv=cv_sets)

    # Fit the grid search object to the data to compute the optimal model
    grid = grid.fit(X, y)

    # Return the optimal model after fitting the data
    return grid.best_estimator_

No, I would not use this model in a real-world setting. The data is 40 years old, which means the predictions made are suitable only for the specific economic conditions in Boston in 1978. Inflation is important in that \$1 in 1978 is worth more than \$1 today. Thus, even for the client with the worst home, a prediction of \$237,478.72 understates the value of their home today.

Also, changing consumer tastes for houses and market conditions can change over time. Perhaps people today prefer to live in condos resulting in an increase in competition facing people selling their homes. This may depress inflation-adjusted prices.

Another issue is in making predictions for houses that would be considered outliers in the training data. The training sample does not contain enough data on exteremely poor and rich neighborhoods.
